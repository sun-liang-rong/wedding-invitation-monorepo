# 婚礼电子请帖系统 - 实现计划

## 目标
实现类似「婚礼纪」的婚礼请帖模板系统，采用数据驱动方式，通过 Renderer 动态渲染模板。

## 实施步骤

### 步骤 1：定义 WeddingData 数据 Schema
- 新郎/新娘信息（姓名、头像、介绍）
- 婚礼时间（日期、时间）
- 地点信息（名称、地址、经纬度）
- 封面图片
- 相册图片数组
- 祝福文案
- 宾客签到信息

### 步骤 2：定义 Template Schema
- 模板元信息（ID、名称、描述、作者）
- 主题配置（主色、辅色、字体、背景）
- 页面配置（布局、间距、圆角）
- Sections 数组
  - 每个 section 包含：type、style、props
  - 支持 section 嵌套

### 步骤 3：实现占位符解析引擎
- 支持 `{{data.path.to.value}}` 语法
- 支持默认值 `{{data.name||默认值}}`
- 支持条件渲染 `{{data.showFeature?true:false}}`
- 支持循环渲染 `{{#each data.gallery}}`

### 步骤 4：实现 Section 组件库
- SectionHeader（标题区）
- SectionCover（封面区）
- SectionCouple（新人信息区）
- SectionEvent（婚礼详情区）
- SectionGallery（相册区）
- SectionBlessing（祝福区）
- SectionGuestbook（留言板）

### 步骤 5：实现核心 Renderer 组件
- 解析模板 JSON
- 递归渲染 sections
- 注入数据上下文
- 处理响应式样式

### 步骤 6：创建示例模板
- 参考婚礼纪风格的"浪漫与你"模板
- 包含完整的 JSON 配置
- 适配移动端布局

### 步骤 7：创建使用示例页面
- 展示如何在 Next.js 中使用
- 包含真实数据注入示例

## 目录结构

│   ├── wedding-data.ts      # WeddingData Schema

├── components/

│   │   ├── index.tsx        # Renderer 主组件
│   │   ├── section-factory.tsx  # Section 工厂
│   │   └── placeholder.tsx  # 占位符解析器
│   └── sections/            # Section 组件库
│       ├── SectionHeader.tsx
│       ├── SectionCover.tsx
│       ├── SectionCouple.tsx

## 输出内容

1. **types/wedding-data.ts** - WeddingData 类型定义
2. **types/template.ts** - Template Schema 类型定义
3. **components/renderer/index.tsx** - 核心 Renderer 组件
4. **components/renderer/section-factory.tsx** - Section 工厂
5. **components/renderer/placeholder.ts** - 占位符解析器
6. **sections/*.tsx** - Section 组件库
7. **templates/romantic-you.json** - 示例模板
